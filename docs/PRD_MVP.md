# PRD – MVP（Phase 2-first「通話→予定化テンプレ」）

## 戦略転換：Phase 2-first

**従来**：Phase 1（.icsのみ）→ Phase 2（通話追加）  
**新戦略**：Phase 2-first（通話テンプレから開始）

**理由**：
- 需要が強い（病院予約・飲食キャンセル・再配達は「電話したくない」痛みが強い）
- 差別化が明確（.icsだけでは他のカレンダーアプリと差別化できない）
- 価値の即時性（通話代行は「6分節約」が体感しやすい）
- Action Cloudへの布石（通話テンプレの実績が外販の信頼になる）

**.icsは「フォールバック」として常時待機**（権限未連携/外部障害時でも即価値）

---

## ペルソナ（初期）
- 25–40歳、忙しくて「電話したくない/先送り」が日常的課題
- 病院予約・飲食店キャンセル・再配達などの「電話が面倒」な用事を抱えている

## ユースケース（初期テンプレ3本）

### 1. 病院予約（一次診療）
**フロー**：
1. ユーザー：「明日10:30に◯◯クリニック予約」（7秒入力）
2. **Call Consent**：通話開始の承認（10分有効）
3. `call.place`：AIが電話で予約（空き枠確認→予約）
4. `call.summary`：要約生成（「明日10:30で予約確定。受付:田中様」）
5. **PlanA/B**提示：
   - A: カレンダー追加 + 家族へ共有 + 出発リマインド
   - B: カレンダー追加のみ
6. **Confirm once**：1タップで並列実行

**期待値**：
- 通話成功率：≥85%（Year 1）
- vMB：6-10分（電話待ち時間＋カレンダー入力）
- FEA：3件（電話/カレンダー/メッセ）

---

### 2. 飲食キャンセル/リスケ
**フロー**：
1. ユーザー：「今日19時の予約をキャンセルして、明日19時に変更」（7秒入力）
2. **Call Consent**：通話開始の承認
3. `call.place`：AIが電話でキャンセル＋再予約
4. `call.summary`：要約生成（「今日キャンセル完了。明日19時で予約確定」）
5. **PlanA/B**提示：
   - A: カレンダー更新 + 相手へ連絡 + リマインド
   - B: カレンダー更新のみ
6. **Confirm once**：1タップで並列実行

**期待値**：
- 通話成功率：≥90%（飲食店は対応が早い）
- vMB：5-8分
- FEA：3件

---

### 3. 再配達/配送業者対応
**フロー**：

**期待値**：

- FEA：2件

## 受け入れ基準（必須）

### Call Consent（通話の承認）
- ✓ **Call Consent**（/api/approve, TTL 10分）なしの通話は送出しない（**400_CALL_CONSENT_REQUIRED**）
- ✓ `approve_id`が欠落している場合は受け付けない

### 通話品質
- ✓ `call.status` 初回通知 ≤ **5s**
- ✓ `call.summary` 終話から ≤ **10s**
- ✓ 通話成功率 ≥ **90%**（p50）

### Plan生成
- ✓ 要約から **PlanA/B**を生成
- ✓ **Confirm once**で**2アクション以上**を並列実行
- ✓ **部分成功**でも全体完了扱い（失敗分はリトライ/再提案）

### 監査・安全性
- ✓ **Undo 10秒** / **監査ログ（Execution Ledger）** / **Idempotency-Key（24h）**必須
- ✓ **発信者番号**は非通知不可・再発信時は同一番号、冒頭で「**ユーザー依頼の連絡**」を明示
- ✓ **.icsフォールバック**は常時有効（権限未連携/外部障害時）

### KPI（p50）
- ✓ 提案表示≤1.5s
- ✓ 通話成功≥90%
- ✓ 誤実行<0.5%
- ✓ vMB中央値≥6分
- ✓ Screen-off≥70%

---

## 付録：Fallback（.icsのみ）最小要件

**.icsは「フォールバック」として常時有効**

### 発動条件
- 権限未連携（カレンダー/メッセージ）
- 外部API障害（通話プロバイダ/カレンダーAPI）
- ユーザーが「電話しない」を選択

### 動作
- 7秒入力→**2提案**→**.ics生成DL** p50≤1.5s
- duration_min / slot 表示
- 静音テキストへ自動フォールバック
- PIIは保存せず、要約+操作メタのみを長期保持
