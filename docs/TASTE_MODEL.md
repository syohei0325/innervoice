# TASTE MODEL – 好み学習

## 目的
ユーザーの好み（系統/価格/雰囲気/照度/席/駐車/距離/混雑/時間帯）を学習し、提案の**Top‑1採択率**を向上させる。

## 特徴量
- **系統**：和食/洋食/中華/イタリアン/フレンチ/エスニック
- **価格**：ランチ予算/ディナー予算/上限
- **雰囲気**：静か/賑やか/カジュアル/フォーマル
- **照度**：明るい/暗い/自然光
- **席**：カウンター/テーブル/個室/テラス
- **駐車**：必須/不要/近隣可
- **距離**：徒歩圏/車で15分/30分
- **混雑**：空いている/適度/賑わい
- **時間帯**：ランチ/ディナー/深夜

## 学習データ
- **承認/却下**：提案に対する反応（👍/👎）
- **再訪**：同じ店への再訪頻度
- **滞在時間**：長い=満足、短い=不満
- **即時反応**：提案から承認までの時間（短い=高評価）
- **理由タグ**：却下時の理由（高い/遠い/混雑/暗い/駐車×…）

## 推論
1. **端末先行**：ローカルで初期ランキング（プライバシー優先）
2. **サーバ再ランク**：要約＋統計のみで精緻化（本文は送らない）

## UI
- **却下時の理由タグ**：1タップで収集（高い/遠い/混雑/暗い/駐車×…）
- **Why‑this‑for‑you**：提案理由を表示（最大3理由＋出典＋信頼度）

## KPI
- **Top‑1採択率** ≥ 55%
- **編集距離** ≤ 20%（提案から実際の選択までの差）
- **Time‑to‑Confirm** p50 ≤ 3秒
- **Taste命中率** ≥ 40%（3ヶ月後）

## プライバシー
- 本文は保存しない（要約＋統計のみ）
- 端末優先（PDV: Personal Data Vault）
- エクスポート/削除可能
